# 개발 워크플로우

## 🔒 머지 룰

- `main` 브랜치에 직접 push 불가
- 모든 PR은 최소 1명 리뷰 필요
- Lint/Build 통과 필수
- 머지 방식은 Squash 머지 사용

## ⚙️ 작업 플로우

1. 이슈 생성 및 할당
2. `프로젝트명/이슈번호/기능명` 브랜치 생성
3. PR 생성 + 자동 연결 (`Closes #번호`)
4. 리뷰 & 머지 후 main 브랜치에 반영

## 📝 커밋 메시지 컨벤션

- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 스타일 변경 (포맷팅, 세미콜론 누락 등)
- refactor: 코드 리팩토링
- test: 테스트 코드 추가/수정
- chore: 빌드 프로세스, 패키지 매니저 설정 등
